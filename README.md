# Quant X SmartMoney

SmartMoney capital rotation dashboard for sectors, AI themes, and Physical AI themes.

The app estimates capital rotation from OHLCV data, aggregates the signal by industry
or theme, highlights unusual rotation changes, and renders an interactive Streamlit
dashboard.

## Requirements

- uv
- Python 3.12

## Setup

Install the locked project environment:

```bash
uv sync --locked
```

## Local Development

Run the Streamlit dashboard:

```bash
uv run streamlit run app.py
```

`uv run` automatically uses the project `.venv`. If you prefer to activate the
environment explicitly in your shell, you can run:

```bash
source .venv/bin/activate
streamlit run app.py
```

Open:

```text
http://localhost:8501
```

Generate the static HTML report:

```bash
uv run python cli.py
```

Prefetch market data and rebuild the dashboard cache:

```bash
uv run python scripts/prefetch_data.py
```

## Data

- `taxonomy.yaml` defines sectors, industries, AI themes, Physical AI themes, and tickers.
- `data/cache.pkl` is the precomputed dashboard cache.
- If `data/cache.pkl` exists, the Streamlit app loads it first.
- If the cache is missing or unreadable, the app falls back to live yfinance fetches.

## Production Data Update

GitHub Actions runs `.github/workflows/daily_data.yml` after US market close on weekdays.

The workflow:

1. Installs uv.
2. Runs `uv sync --locked`.
3. Runs `uv run python scripts/prefetch_data.py`.
4. Validates `data/cache.pkl`.
5. Commits the updated cache back to the repository.

## Project Layout

- `app.py`: Streamlit dashboard.
- `cli.py`: command-line static HTML report generator.
- `smartmoney/rotation.py`: capital rotation calculations.
- `smartmoney/loader.py`: cached and live data loading for the dashboard.
- `smartmoney/data.py`: yfinance OHLCV fetching and weekly resampling.
- `smartmoney/taxonomy.py`: taxonomy loading helpers.
- `smartmoney/report.py`: self-contained HTML report renderer.
- `scripts/prefetch_data.py`: scheduled cache builder.
- `taxonomy.yaml`: source of truth for sector/theme ticker groups.
- `data/cache.pkl`: precomputed dashboard data generated by the prefetch workflow.

## Notes

- uv is the preferred project environment; `uv.lock` should stay committed.
- `requirements.txt` is kept for compatibility with older workflows.
- The `market_cap` weighting currently uses dollar volume, `close * volume`, as a
  liquidity proxy. It is not true market capitalization weighting.
- Do not edit `data/cache.pkl` manually. Rebuild it with `uv run python scripts/prefetch_data.py`.
